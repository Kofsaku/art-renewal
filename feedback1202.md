## ■ プロジェクト背景

- アート社は入退室管理システムのセキュリティ専門メーカー
- 既存の主流ソフト「XSS」からWebソフト化への移行を推進
- 直感的操作と時代変化に対応したシンプルなデザインを目指す
- 内部設計はXSS製作会社、画面デザインはSET様指定形式で納品

## 【画面①】個人情報一覧

### ● 決定した仕様

1. **送信ボタンの追加**
   - 画面に「送信」ボタンを新設

2. **ボタン追加**
   - 「入退室制限時間帯の管理」ボタン
   - 「通行権限の一括変更」ボタン

3. **フィルター実行中の可視化**
   - フィルター適用中であることが視覚的に分かるようにする
   - 黒三角アイコンなど、表現方法は開発側に委任

4. **フィルター後の人数表示**
   - 「現条件：●人」のように該当人数を表示

5. **履歴遷移の仕様変更（重要）**
   - 行右側の履歴ボタンは**削除**
   - 該当行を**右クリック** → ポップアップ表示 → 履歴画面へ遷移
   - ポップアップ内の選択項目：
     - **期間**: 当日 / 前日～ / 期間指定（複数選択不可）
     - **履歴種類**: 全て / 軽エラー / 重エラー / 重エラー復旧

6. **左上のボタン変更**
   - 「新規登録」「削除」「消去」に統一
   - ※削除/消去は運用上どちらも必要

7. **編集操作**
   - 該当行のダブルクリックで編集画面を開く

## 【画面②】個人情報 登録/編集

### ● 主な仕様決定

1. **個人コード**: 選択ボタン削除（入力オンリー）
2. **期間入力**: 手入力 + カレンダー入力の両方に対応
3. **代替コード**: 選択ボタン削除（入力オンリー）
4. **ログインパスワード**: 削除（別機能に移行）
5. **通門モード**: 以下4項目のチェックボックスに変更
   - (a) 読取禁止（デフォルト：有効）
   - (b) アンチパス（デフォルト：有効）
   - (c) 警備セット操作不可（デフォルト：無効）
   - (d) 監視カード（デフォルト：無効）
6. **通行権限**: ダミーデータを追加して表示サンプルとする
7. **ボタン文言**: 「登録」→「登録送信」に変更
8. **生体INDEX設定**: 他の項目を削除し「バイオID」のみ残す。その下に「通門モード」セクションを追加
9. **空白部削除**: レイアウトを調整し、空白部をなくす
10. **文字・枠サイズ**: 小さすぎないように。最大桁数の1.5倍程度の余裕を確保

### ● レイアウト構成

必須項目 → 利用期間設定 → 代替設定 → 生体INDEX設定 → 通門モード → 顔写真 → 通行権限

### ● 必須項目（Enter押下時の挙動）

| No | 項目 | 未入力時Enter押下の挙動 |
|----|------|-------------------------|
| 1 | 個人コード | 未入力不可 |
| 2 | 発行回数 | スキップ |
| 3 | 管理番号 | 個人コード（桁数合わせ）を自動入力 |
| 4 | 氏名 | 個人コードを自動入力 |
| 5 | 氏名(カナ) | 個人コードを自動入力 |
| 6 | テナント | スキップ |
| 7 | 所属 | 「000」（桁数合わせ）を自動入力 |
| 8 | 区分 | 「000」（桁数合わせ）を自動入力 |
| 9 | テンキー暗証番号 | 「0000」を自動入力 |
| 10 | 利用開始日 | 「9999/99/99」を自動入力 |
| 11 | 利用終了日 | 「9999/99/99」を自動入力 |

### ● 任意項目（デフォルトはグレーアウト）

1. 代替コード
2. 代替コード利用終了日
3. バイオコード
4. 通門モード（4項目）

### ● 対応分担

- **miitaso**: 次項目への移行対応（Enter押下時のフォーカス移動）
- **SET様**: 自動入力の対応

## 【画面④】報告書（過去の履歴データ確認）

### ● 運用イメージ（完全版）

1. **初期状態**: 報告書画面を開く → 履歴データ「0件」の状態で表示
2. **期間選択**: 「抽出」ボタンを押して対象期間を選択 → 履歴データ「n件」が表示される  
3. **データ絞り込み**: 各項目フィルターから履歴データをさらに絞り込む
4. **期間確認**: 画面上部に現在の対象期間が常時表示される
5. **件数確認**: 右上に現在の条件での件数が常時表示される

### ● 必須機能（3つの要望）

#### **【要望①】抽出ボタン追加** ※必須機能・仮配置
- **配置**: 条件保存・エクスポートセクションの下に「抽出」ボタンを配置
- **機能**: ボタンを押すと対象期間選択ポップアップが表示される
- **選択後**: 選択した期間のデータが一覧に表示される
- **UI**: オレンジ色のボタンで目立つように配置

#### **【要望②】対象期間表示** ※必須機能・仮配置
- **表示位置**: 画面上部に「対象期間 ※2025/10/10の場合」として表示
- **表示形式**: 
  - ラベル: 「現在の指定期間：」
  - 期間表示: 「2025/9/11 0:00 ～ 2025/10/10 23:59」形式
  - **文字色**: 赤字で強調表示
- **表示タイミング**: 抽出ボタンで期間選択後に更新される

#### **【要望③】現条件での件数表示** ※必須機能・仮配置
- **表示位置**: 画面右上
- **表示形式**: 「現条件：●件」
- **更新タイミング**: 
  - 期間選択後
  - フィルター適用後
  - データ変更時

### ● 詳細仕様

#### **1. 抽出ボタン仕様**
- **配置位置**: 条件保存・エクスポートセクションの下に配置
- **ポップアップ選択肢**: 
  - 全期間
  - 期間指定
- **選択制限**: 複数選択不可（ラジオボタン形式）
- **期間指定時**: 開始日時・終了日時を個別に設定可能

#### **2. 対象期間表示仕様**
- **機能**: 現在の指定期間を表示するのみ（選択機能なし）
- **初期状態**: 「未選択」または空白表示
- **選択後**: 具体的な日時範囲を表示
- **フォーマット**: YYYY/M/D H:MM形式
- **色指定**: 赤字（#FF0000または類似色）
- **位置**: 画面上部、抽出ボタンの右側エリア

#### **3. 件数表示仕様**
- **初期状態**: 「現条件：0件」
- **データ表示後**: 実際の件数を表示
- **位置**: 画面右上角
- **更新**: リアルタイム更新

#### **4. 既存機能との連携・削除対象**
- **期間選択ラジオボタン**: 完全廃止（抽出ボタンに機能統合）
- **削除必須**: 「1ヶ月前」「1週間前」「1年前」「1日前」のラジオボタンは画面から完全削除
- **対象期間セクション**: 既存の対象期間選択UIは全て削除し、表示のみに変更
- **フィルター**: 既存のExcel型フィルターと連携
- **ダミーデータ**: 対応可能（開発側確認済み）

### ● 技術要件・UI削除指示
- **複数選択**: 期間選択は単一選択のみ
- **データ連携**: 抽出→表示→フィルターの順次処理
- **UI応答性**: ボタン押下から表示まで3秒以内
- **重要**: 画面上の「1ヶ月前」「1週間前」「1年前」「1日前」のラジオボタンは必ず削除
- **対象期間UI**: 選択機能を全て削除し、現在設定値の表示のみに変更

## 【画面⑤】ステータスモニター（リスト上でゲート状態確認）

### ● 決定した仕様

1. **ゲート情報表示**
   - ゲート番号と名称を表示
   - 64件表示時も文字が小さくなりすぎないように配慮

2. **アイコン変更**
   - サンプルアイコン（扉が開いている画像等）を反映

3. **操作方法変更**（履歴/遠隔ボタンを廃止）
   - **枠内クリック** → 遠隔操作ポップアップ
     - 選択肢: 連続解錠 / 解錠 / 施錠 / 警備セット / 警備セット解除（複数選択不可）
   - **枠内右クリック** → 拡張メニュー表示
     - 報告書作成など（拡張性を持たせる）
     - 期間・履歴種類選択 → 履歴画面へ遷移

4. **各状態フィルター**
   - ON/OFFで表示切替（枠は移動せず、表示/非表示で切替）

### ● 表示仕様

- **件数選択**: 16件 / 32件 / 64件の切り替え可能
- **枠サイズ**: フルHDで全数表示できる最大サイズを確保
- **ブラウザサイズ対応**: 1枠サイズは固定、表示列数が変化（残りはスクロール表示）

---

## 【画面⑥】マップモニター（マップ上でゲート状態確認）

### ● 決定した仕様

1. **欄名変更**: 「ゲート欄」→「フロア一覧」（フロア1、フロア2、フロア3等）

2. **一斉制御ボタン追加**
   - グループ選択（複数選択可）と遠隔操作が可能なポップアップを表示
   - グループ選択: グループA / グループB / グループC...（複数選択可）
   - 遠隔操作: 連続解錠 / 解錠 / 施錠 / 警備セット / 警備セット解除（複数選択不可）

3. **サンプルマップ**: 共有済みの黒背景マップ画像を反映

4. **アイコン配置**: 開発側に委任（動作が分かればOK）

5. **アイコンクリック**: 遠隔操作ポップアップを表示

6. **アイコンホバー**: ゲート状態（ステータス情報）をツールチップ表示

### ● 表示仕様

- **縦横比**: 常に固定（変化しない）
- **サイズ変化**: ブラウザサイズに応じて拡大縮小
- **最低サイズ**: 800×600（これより小さくならない、残りはスクロール表示）
